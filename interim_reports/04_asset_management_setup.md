# 중간 보고서: 정적 자산 관리 체계 구축

## 1. 개요
프로젝트의 확장성과 유지보수성을 높이기 위해 `public` 폴더 내 정적 자산(이미지, 로고, 아이콘 등)의 관리 체계를 구축하였습니다.

## 2. 수행 내용
- **폴더 구조 재설계**: 용도별로 명확히 구분된 하위 폴더 시스템을 생성하였습니다.
    - `public/images/hero/`: 히어로 섹션 전용 이미지
    - `public/images/common/`: 공통 사용 이미지
    - `public/logos/`: 서비스 및 외부 로고 (Next.js, Vercel 로고 이동 완료)
    - `public/icons/`: 각종 아이콘 파일 (기본 SVG 아이콘 이동 완료)
    - `public/samples/`: 샘플 이미지 및 더미 데이터 자산
- **기존 자산 정리**: 루트 수준에 흩어져 있던 기본 SVG 파일들을 각각 `logos`와 `icons` 폴더로 이동하여 정리하였습니다.

## 3. 향후 계획
- 서비스 로고(`logos`) 및 메인 비주얼(`images/hero`) 실제 자산 반영
- `next/image` 컴포넌트를 활용한 성능 최적화 적용
